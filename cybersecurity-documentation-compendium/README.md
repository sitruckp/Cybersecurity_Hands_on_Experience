# Monmouth University Cybersecurity Documentation Portfolio

**Author:** Kevin P. Curtis  
**Role:** Instructional Designer / Cybersecurity Practitioner  
**Certifications:** ISC² CC, Google Cybersecurity Analyst, IBM Cybersecurity Analyst, IBM Ethical Hacking (pursuing SSCP)

## Purpose
This repository contains documentation I developed following a ransomware incident to strengthen the University's cybersecurity posture. All materials are practical, audit-ready, and aligned with NIST, CIS, and industry best practices.

## Documents Included

- **180-day-phased-soc-plan.md** — Full phased roadmap to build an internal SOC
- **90-day-soc-implementation-plan.md** — Quick-start Blue Team capability
- **soc-charter.md** — Official SOC Charter
- **soc-policies-and-procedures.md** — Audit-ready SOC policies
- **incident-response-plan.md** — Full Incident Response Plan (IRP)
- **incident-response-playbooks.md** — Ready-to-use playbooks
- **incident-evidence-insurance-checklist.md** — Evidence handling & insurance checklist
- **proposed-rules-of-engagement-roE.md** — Safe testing boundaries
- **executive-summary-business-case-internal-soc.md** — Business case for internal SOC

## Key Features
- Blue Team first, observe-only approach
- Strong alignment with NIST CSF 2.0 and NIST SP 800-61
- Designed for higher-education operational realities
- Audit-ready and insurance-friendly
